# Проверка на вирусы и вредоносное поведение

## Итог

По статическому анализу исходников признаков вируса, трояна, стиллера, дроппера или бэкдора я не нашёл.

Проект агрессивный по доступам, но логика соответствует его заявленной роли anti-cheat/screenshare utility:

- чтение памяти процессов;
- low-level hooks мыши и клавиатуры;
- чтение WMI, реестра, event logs;
- вызовы `fsutil` и экспорт текстовых артефактов;
- проверка цифровых подписей исполняемых файлов.

## Что я искал специально

- сетевую эксфильтрацию;
- скрытые загрузки полезной нагрузки;
- автозапуск/постоянство;
- инъекции в чужие процессы;
- подмену реестра и установку сервисов;
- скрытое удаление пользовательских файлов.

## Что нашёл

### Не найдено

- `CreateRemoteThread`
- `WriteProcessMemory`
- `VirtualAllocEx`
- `NtCreateThreadEx`
- `URLDownloadToFile`
- HTTP/WinINet-код для эксфильтрации
- `CreateService` / `DeleteService`
- `RegSetValue` / `RegCreateKey`
- логика persistence/autostart

### Найдено, но ожидаемо для anti-cheat утилиты

- `OpenProcess(PROCESS_VM_READ...)`
- `ReadProcessMemory`
- `SetWindowsHookEx`
- `WinVerifyTrust`
- `CreateProcessW` для внешнего memory scanner
- `fsutil usn readjournal`

## Практический вывод

Исходники выглядят как инструмент локального аудита/форензики, а не как malware. При этом:

- у него высокий уровень доступа;
- он читает чувствительные системные артефакты;
- его всё равно нужно запускать только в доверенной среде;
- внешний `memory.exe` нужно проверять отдельно, потому что он не лежит в этом репозитории.

## Отдельные замечания по безопасности

### 1. Неодинаковое имя внешнего бинарника

Оригинальный проект местами ищет `memory.exe`, а местами `memory scanner.exe`. Это не вирусный признак, а баг сопровождения.

### 2. Windows Defender

Я попытался выполнить штатную проверку каталога через Defender (`Start-MpScan` и `Get-MpComputerStatus`), но система вернула общую ошибку Defender и не дала завершить scan.

Следствие:

- статический кодовый аудит выполнен;
- “антивирусно чисто” на 100% через локальный Defender в этой среде подтвердить не удалось;
- для окончательной уверенности стоит отдельно проверить внешний `memory.exe` и готовый `rust_project.exe` уже на вашей машине через Defender/VT/Hybrid.
