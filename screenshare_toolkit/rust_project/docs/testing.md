### Дополнительные проверки

- отдельная попытка прогнать локальный Windows Defender scan.

## Что покрыто unit tests

- извлечение `VID/PID`;
- нормализация `file:///` путей из памяти explorer;
- выделение Windows-путей из строк memory scanner;
- распознавание split-extension pattern;
- детекция non-ASCII строк.

## Ограничения тестовой среды

- Внешний `memory.exe` отсутствует в репозитории, поэтому memory-based runtime checks не гонялись end-to-end.
- `Start-MpScan`/`Get-MpComputerStatus` в этой среде не завершились успешно из-за ошибки Defender.
- Часть Windows forensic checks корректно проверяется только при запуске от администратора на реальной пользовательской системе.

## Что проверить уже у вас после сборки release

1. Положить рядом с exe внешний `memory.exe` или `memory scanner.exe`.
2. Запустить `rust_project.exe` от администратора.
3. Пройти пункты 1, 2, 3 и 4 из меню.
4. Проверить содержимое `scan_exports`.
